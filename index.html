Semantic Non-Semantic

Semantic Tags

Semantic tags are those that provide meaning to the content.

  1. <header> – Represents the introductory content of a page or section, like a logo or navigation.
  2. <nav> – Contains navigation links for the website or a section.
  3. <main> – Represents the main content of the page that is unique and central.
  4. <section> – Groups related content together, like a chapter or topic.
  5. <article> – Represents an independent, self-contained piece of content, like a blog post or news article.
  6. <aside> – Contains content related to the main content, like a sidebar or additional info.
  7. <footer> – Represents the footer of a page or section, typically contains copyright or contact info.
  8. <figure> – Self-contained content like an image, chart, or code snippet.
  9. <img> – Represents an image. Use the alt attribute to describe the image for accessibility.
  10. <figcaption> – Provides a caption for a figure element.
  11. <em> – Emphasized text (usually italic), conveys importance.
  12. <strong> – Strong importance (usually bold), conveys meaning.
  13. <mark> – Highlights text.
  14. <time> – Represents a date or time.
  15. <cite> – Represents the title of a work or citation.
  16. <code> – Represents inline code snippets.
  17. <kbd> – Represents user input, like keyboard keys.
  18. <samp> – Represents sample output from a program.
  19. <address> – Contact information for the author/owner of a document.
  20. <details> – Represents additional details the user can view or hide.
  21. <summary> – Summary or caption for a details element.
  22. <dialog> – Represents a dialog box or interactive component.

Non-Semantic Tags

Non-semantic tags do not provide meaning; they are used for layout or styling purposes.

  • <div> – A generic block-level container used for layout or grouping elements.
  • <span> – A generic inline container used for styling parts of text or inline content.
  • <b> – Bold text for styling only. No semantic meaning.
  • <i> – Italic text for styling only. No semantic meaning.
  • <u> – Underlined text for styling only.
  • <small> – Smaller text for styling purposes.

Special / Self-Closing Tags

Self-closing tags, also called void elements, do not have a closing tag. They are used for elements that don’t have content inside them.

  1. <img> – Inserts an image.
  2. <br> – Inserts a line break.
  3. <hr> – Inserts a horizontal rule.
  4. <input> – Creates an input field.
  5. <meta> – Defines metadata like charset or viewport.
  6. <link> – Links to external resources like CSS files.

Unlike normal tags, these tags do not wrap content and do not require a closing tag.
